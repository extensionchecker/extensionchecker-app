# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest on `main` | Yes |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Instead, use one of these channels:

1. **GitHub Security Advisories** (preferred): Report privately via
   [GitHub's security advisory feature](../../security/advisories/new).
   This keeps the report confidential until a fix is available.

2. **Email**: Send details to **security@extensionchecker.org**.

### What to include

- A description of the vulnerability and its potential impact.
- Steps to reproduce or a proof-of-concept.
- The affected component(s) and version(s), if known.

### What to expect

- **Acknowledgment** within 48 hours of your report.
- We will work with you to understand and validate the issue.
- We aim to release a fix promptly and will coordinate disclosure with you.
- Credit will be given in the release notes unless you prefer to remain
  anonymous.

## security.txt

A machine-readable [`security.txt`](https://securitytxt.org/) is served at
`/.well-known/security.txt` on the public deployment. It points to this same
disclosure process.

## Scope

This policy covers the ExtensionChecker application code in this repository.
It does not cover third-party dependencies - please report those to their
respective maintainers.
